## AI 助手行为指南

当用户在新窗口中打开此项目时，请：

1. 先读取此文件了解当前进度
2. 检查 `当前进度` 表格，找到第一个状态为 `⏳` 或 `🔜` 的步骤
3. 询问用户是否继续该步骤
4. 每完成一个步骤，更新此文件中的进度表格
5. 教学风格（交互式教学模式 — 必须遵守）：
   - 语言约定：对话使用中文，生成的代码和注释使用英文
   - 利用用户的 C/C++ 经验做对比讲解
   - 每个概念都附带可运行的代码示例
   - 代码放在 `src/` 目录下，按阶段组织
   - **交互式教学流程**：
     a. 先讲解单个概念（对比 C++ 差异），简短精炼
     b. 给出一小段代码，让用户自己写入文件
     c. 让用户自行编译运行，观察输出结果
     d. 运行成功后问用户几个理解确认问题
     e. 用户回答后再进入下一个概念
   - **禁止行为**：
     - 禁止一次性创建完整的大文件让用户被动阅读
     - 禁止代替用户运行编译命令（除非用户明确要求帮忙）
     - 禁止连续讲解多个概念而不暂停等待用户反馈
   - 每个教程文件内容控制在 30-80 行，聚焦单一概念

## C/C++ vs Java 速查表（持续更新）

| 概念 | C/C++ | Java |
|------|-------|------|
| 入口函数 | `int main()` | `public static void main(String[] args)` |
| 编译 | `gcc/clang` → 原生二进制 | `javac` → 字节码(.class) → JVM 执行 |
| 内存管理 | `malloc/free`, `new/delete` | 自动垃圾回收(GC) |
| 指针 | 支持指针运算 | 无指针，只有引用 |
| 多继承 | C++ 支持 | 不支持（用接口替代） |
| 字符串 | `char*`, `std::string` | `String`（不可变对象） |
| 布尔类型 | `bool` (C++), `int` (C) | `boolean` |
| 数组 | `int arr[10]` | `int[] arr = new int[10]` |
| 命名空间 | `namespace` | `package` |
| 头文件 | `#include` | `import` |
| 操作符重载 | 支持 | 不支持 |
| 析构函数 | `~ClassName()` | `finalize()`（不推荐用） |
