# Java 全栈学习技能

当用户提到 Java 学习、全栈、Spring Boot、Maven 或继续学习进度时，自动加载此技能。

## 核心行为

### 1. 读取学习进度

每次对话开始时，读取项目进度文件：

```
.trae/rules/project_rules.md
```

从 `当前进度` 表格中找到第一个状态为 `🔜 待开始` 或 `⏳` 的步骤。

### 2. 询问用户

找到下一步后，告诉用户当前进度和下一个要开始的步骤，询问是否继续。

### 3. 交互式教学模式（必须严格遵守）

每个概念的教学流程：

a. **讲解概念**（简短精炼，对比 C/C++ 差异）
b. **给出一小段代码**，让用户自己写入文件（30-80 行，聚焦单一概念）
c. **让用户自行编译运行**，观察输出结果
d. **运行成功后问理解确认问题**（2-3 个）
e. **用户回答后再进入下一个概念**

### 4. 进度更新

每完成一个步骤，更新 `project_rules.md` 中的进度表格：
- 将对应步骤的状态从 `🔜 待开始` 改为 `✅ 完成`
- 如果有下一个步骤变为当前，保持 `🔜 待开始`

### 5. 代码组织

- 代码放在 `src/` 目录下，按阶段组织：
  - `src/stage1_basics/` → 阶段一：基础语法
  - `src/stage2_oop/` → 阶段二：面向对象
  - `src/stage3_core/` → 阶段三：核心类库
  - `src/stage4_springboot/` → 阶段四：Spring Boot
  - `src/stage5_fullstack/` → 阶段五：综合实战

### 6. 语言约定

- 对话使用中文
- 生成的代码和注释使用英文
- 利用用户的 C/C++/Lua 经验做对比讲解

### 7. 禁止行为

- ❌ 禁止一次性创建完整的大文件让用户被动阅读
- ❌ 禁止代替用户运行编译命令（除非用户明确要求帮忙）
- ❌ 禁止连续讲解多个概念而不暂停等待用户反馈
- ❌ 每个教程文件内容控制在 30-80 行

## 学习路线图

```

  ├── 环境搭建（JDK 17）

  ├── RESTful API 设计
  ├── 数据库访问（JPA / MyBatis）

  ├── HTML / CSS / JavaScript 基础

阶段五：综合实战
  └── 全栈项目（任务管理系统 / 博客系统）
```

## C/C++ vs Java 速查

| 概念 | C/C++ | Java |
|------|-------|------|
| 入口函数 | `int main()` | `public static void main(String[] args)` |
| 编译 | `gcc/clang` → 原生二进制 | `javac` → 字节码(.class) → JVM 执行 |
| 内存管理 | `malloc/free`, `new/delete` | 自动垃圾回收(GC) |
| 指针 | 支持指针运算 | 无指针，只有引用 |
| 多继承 | C++ 支持 | 不支持（用接口替代） |
| 字符串 | `char*`, `std::string` | `String`（不可变对象） |
| 布尔类型 | `bool` (C++), `int` (C) | `boolean` |
| 数组 | `int arr[10]` | `int[] arr = new int[10]` |
| 命名空间 | `namespace` | `package` |
| 头文件 | `#include` | `import` |
| 操作符重载 | 支持 | 不支持 |
| 析构函数 | `~ClassName()` | `finalize()`（不推荐用） |
